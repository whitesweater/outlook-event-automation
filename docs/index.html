Outlook Event Automation

A server-side agent that reads event-heavy inboxes, extracts structured calendar data, writes eligible events, and keeps a clear audit trail for ignored and ambiguous mail.

Pipeline

Read

Outlook Mail and Gmail sources are supported. The active production path uses Microsoft Graph.

Extract

Messages are processed in batches with schema-constrained Responses API output and split-retry fallback.

Write

Outlook Calendar or Google Calendar can receive eligible events, with dedupe keys preventing repeat writes.

Safety

Conservative States

  • ignored: not a calendar event.
  • needs_review: event-like but ambiguous.
  • created: written to calendar.

Source Context

Outlook events are created with an HTML body containing the AI summary, source metadata, and original email text.

Noise Filters

Daily Event Alert summaries, cancellation notices, and recall messages are kept out of automatic writes.

Deploy

$ cp config.example.json config.local.json
$ cp .env.example .env
$ python3 event_agent.py --config config.local.json auth-microsoft
$ python3 event_agent.py --config config.local.json run --source outlook --sink none --limit 20 --force
$ python3 event_agent.py --config config.local.json serve --write
